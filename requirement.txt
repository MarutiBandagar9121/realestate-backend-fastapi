Property Hierarchy
text

    └── Building(s) (e.g., "Tower A", "Tower B")
        └── Wing(s) (optional, e.g., "North Wing", "South Wing")

Key Feature: Stacking Visualization
Visual representation of buildings showing which client occupies which unit

Ability to see client presence across floors and wings

Heat maps and visual analytics for client clustering

Support for recommending new locations based on client adjacency

Property Data Complexity
Challenge: Different Attributes per Property Type
Office Space needs:

Cabin count, workstation capacity, meeting rooms

Floor plate efficiency, natural light percentage

IT infrastructure, pantry facilities, security features

Retail Space needs:

Storefront width, signage options, mall position

Foot traffic score, trading hours, demographic data

Anchor tenant information, visibility score

Industrial Space needs:

Floor load capacity, clear height, column spacing

Power supply, crane availability, loading docks

Hazardous material handling, waste disposal

Logistics Space needs:

Warehouse height, truck parking, dock levelers

Cold storage, temperature control, rail access

Truck turning radius, approach road width

Shared Office needs:

Hot desks, dedicated desks, private cabins

Meeting rooms, event space, community manager

Membership plans, amenities, operating hours

Additional Complexity: One Property, Multiple Offerings
The same physical space can be:

Available for lease AND sale simultaneously

Partially offered as shared office while rest is traditional lease

Different units in same building with different transaction types

User Management
Customers (end users) - browse, shortlist, compare properties

Employees - manage listings, handle deals, schedule visits

Advisors - assigned to deals, conduct site visits

Business Processes to Support
1. Property Discovery
Browse by location (city → location → sublocation)

Filter by property type, price, size, amenities

Shortlist and compare properties

View property details with images and specifications

2. Deal Management (CRM)
Track leads through stages

Link deals to properties/units

Assign advisors to deals

Track client communications

3. Site Visits
Schedule visits for clients

Link visits to specific properties/units

Capture feedback and notes

4. Documentation
Upload documents for properties, units, deals, visits

Store brochures, floor plans, legal documents

5. Client Analytics (Future)
Track client presence across buildings

Analyze client clustering by industry

Generate recommendations for expansions/relocations

Visual heat maps of client concentration

Content Management
Articles, videos, case studies for marketing

Service pages with FAQs and steps

Key Confusions to Resolve
Property Type vs. Transaction Type: Need to separate what the asset IS (office, retail) from how it's OFFERED (lease, sale, investment)

Shared Office: Is it a property type or an operating model within office space?

Units vs. Offerings: How to represent that the same unit can have multiple offerings (lease AND investment)

What You're Looking For
A scalable data model that:

Handles diverse property types with their unique attributes

Supports stacking visualization (building hierarchy with spatial data)

Separates physical assets from commercial offerings

Tracks tenant history and movements for analytics

Enables future AI/ML recommendations based on client clustering

Performs well with thousands of properties and millions of units

Is maintainable and clear for developers

Non-Negotiables
Data integrity - enforce required fields per property type

Flexibility - accommodate new property types in future
